⚠️ Тихое применение (Silent Enforcement)

Приложение НЕ блокируется — нет диалогов, нет System.exit(). Результат детекции упаковывается в DeviceData и отправляется по 3 независимым каналам:

Firebase Crashlytics Incognia SDK · sendLoginEvent() POST /courier/device_token

Бэкенд принимает решение — заблокировать, ограничить функциональность или пометить аккаунт. Клиент ни о чём не знает.

Critical

Нет Certificate Pinning для Glovo API

Нет CertificatePinner, кастомного X509TrustManager, нет <pin-set> в network_security_config.xml. Burp Suite работает из коробки без root. Incognia пиннинг реализует — Glovo нет.

Исправление

OkHttp CertificatePinner с SHA-256 хешами для всех production доменов + явный <pin-set> в network_security_config.xml
Critical

FCM токен как device identity — не привязан к железу

«Fingerprint устройства» — FCM push-токен. Его можно скопировать и воспроизвести. compare_device_token сравнивает только строки — без криптографической привязки к hardware.

Исправление

Android Keystore attestation: ключевая пара в secure element, подписывать серверный challenge
High

Root Detection полностью обходится Magisk DenyList + LSPosed

Все 17 Java-проверок тривиально обходятся Magisk + LSPosed. Каждый метод детекции можно захукать и вернуть чистый результат.

Исправление

Перенести критические проверки в NDK. Интегрировать Play Integrity API (hardware-backed attestation)
High

Frida обнаруживается только на порту 27042

frida-server запускается на любом порту: frida-server -l 0.0.0.0:12345. Детекция через new ServerSocket(27042) немедленно становится неэффективной.

Исправление

Сканировать /proc/net/tcp, /proc/self/maps для frida-agent, /proc/self/task/*/comm
High

SMS 2FA — уязвим к SIM swap и SS7

SIM swap или SS7-перехват позволяет получить код без физического доступа к телефону. Полный доступ к аккаунту включая кошелёк.

Исправление

Добавить TOTP (RFC 6238) как альтернативу SMS. Rate limiting на trigger endpoint (per-IP + per-account)
High

Access token не привязан к устройству

Токены в EncryptedSharedPreferences без hardware attestation. Украденный токен с root-устройства работает на любом другом — нет серверной проверки привязки.

Исправление

Keystore attestation для привязки токена. Верификация device binding на сервере при каждом запросе
High

Socket.IO URL из Firebase Remote Config — подмена возможна

Компрометация Firebase → изменение realtime_connection_url → все курьеры подключаются к вредоносному серверу → фейковые заказы, DoS.

Исправление

Хардкодить или криптографически подписывать список допустимых Socket.IO хостов
High

Wallet Room DB не зашифрована

balance, cashoutAmount, payout, history — в незашифрованном SQLite. На root-устройстве читается через adb shell напрямую.

Исправление

SQLCipher для Room DB. Ключ в Android Keystore (hardware-backed)
High

Перенос заказа — возможный угон с заработком

PUT /deliveries/{id}/change/transfer — неясно верифицирует ли сервер что инициатор является текущим владельцем. Потенциальный угон заказа и заработка.

Исправление

Строгая серверная проверка владения. Подтверждение от получателя. Аудит-лог переводов
High

GPS данных двум сторонним компаниям вне смены

Точные координаты (включая домашний адрес) передаются в Sentiance и Incognia без уведомления пользователя. Sentiance строит поведенческий профиль вне задач Glovo.

Исправление

Ограничить GPS передачу строго периодом активной смены
High

Mapbox токен: Base64 + XOR -63 (тривиально декодируется)

10 строк кода получают валидный токен. Позволяет делать неограниченные запросы Maps/Geocoding за счёт Glovo.

Исправление

URL allowlist в Mapbox Dashboard (разрешить только package name Glovo). Ротация токена
Medium

Полный список приложений → Incognia (3rd party)

getInstalledPackages() — все приложения включая банковские, медицинские, политические. Без согласия пользователя. Нарушение GDPR/CCPA.

Исправление

Передавать только security-relevant пакеты. Явное согласие пользователя
Medium

Root detection — один раз при старте

Если Frida подключается после запуска через ADB — повторной детекции нет. Открытое окно для манипуляций с заказами и кошельком.

Исправление

Повторять периодически или перед критическими операциями (принятие заказа, cashout)
Medium

Тихое enforcement = итерационная петля для атакующего

Нет уведомлений о детекции. Атакующий методично уточняет обход, наблюдая за изменениями в доступности заказов.

Исправление

«Canary» ответы — намеренно изменённые для помеченных устройств вместо тишины
Medium

PIN хеширование — неизвестный алгоритм

PIN «хешируется локально», но алгоритм не идентифицирован. Если MD5/SHA-1 без соли — брутфорс 4-значного PIN тривиален (10 000 комбинаций).

Исправление

HMAC-SHA256 с server-generated nonce (challenge-response)
Medium

Auto-accept без дополнительной аутентификации

Режим активируется одним GET запросом. При компрометации аккаунта атакующий включает auto-accept → курьер получает нежелательные заказы.

Исправление

Биометрия или пароль для включения auto-accept. Push при активации
Medium

Magic Link — нет видимого TTL на клиенте

Клиент не отображает время истечения и не инвалидирует предыдущие ссылки. Фишинговая атака имеет большое окно.

Исправление

TTL ≤ 15 минут. Одноразовое. Инвалидация при повторном запросе
Medium

Биометрия — fallback на слабый PIN устройства

Android Biometric API допускает fallback на device PIN/password (может быть 1234). Обход биометрии при физическом доступе.

Исправление

setNegativeButtonText вместо device credential fallback
Medium

Foreground Service — пользователь не может остановить трекинг

Sentiance сервис перезапускается через 500ms при любом завершении. Нарушение GDPR Article 21 — права на возражение против обработки.

Исправление

Пользовательский контроль трекинга. Остановка вне активных смен
Medium

Proxy Phone — параметр callee может не валидироваться

При слабой серверной валидации — звонок на произвольный номер через инфраструктуру Glovo за их счёт.

Исправление

Принимать только зарегистрированный номер клиента для данного delivery_id
Medium

FCM дедупликация — только 10 сообщений в памяти

Replay-атака с задержкой (после 11+ новых сообщений) пройдёт дедупликацию. Сбрасывается при перезапуске.

Исправление

Дедупликация по timestamp+ID, хранение в Room DB
Medium

COD расхождения — поле justification без валидации

Мошеннический курьер может систематически занижать actualAmount с произвольным обоснованием.

Исправление

Автофлагировка аккаунтов с частыми расхождениями. Ручной ревью для крупных сумм
Low

Bearer token в Socket.IO handshake

Исправление

Low

Нет idempotency key при принятии заказа

Исправление

Low
